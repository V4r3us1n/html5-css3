CSS Animation

No iframe abaixo temos um exemplo bem simples: uma div dentro de outra div. A primeira div (div.parent) é maior e abriga a segunda div (div.child), ambas de coloração distintas, ao passar o mouse sobre a div.parent a div.child irá deslocar 100% de seu tamanho para a direita. Esse deslocamento é realizado através da propriedade transform: translateX(100%).

No CSS existem 2 formas de fazer uma animação, através da propriedade transition, muito boa para fazer animações simples, e também tem a propriedade animation que é mais complexa mas ótima para fazer animações mais elaboradas.

Propriedade Transition:

O maior erro que iniciantes cometem é achar que devem colocar a propriedade transition no :hover da div.parent, mas na verdade ela deve ser aplicada na div.child.

Essa propriedade possui os seguintes valores:

  • transition-delay;
  • transition-duration;
  • transition-property;
  • transition-timing-function;

Use apenas transition para fazer um shorthand desses valores.


            .child {
                background-color: red;
                height: 50%;
                width: 50%;
                transition: transform;
            }

            .parent:hover .child {
                transform: translateX(100%);
            }

No pedaço de código acima, dissemos ao transition para animar apenas a opção transform, propriedade essa que só receberá valor quando passarmos o mouse sobre a div.parent, ou seja, a animação só ocorrerá quando o mouse passar sobre a div.parent. Após isso devemos especificar em quanto tempo a animação vai acontecer, o valor pode ser dado em ms e s. Veja:

Mesmo que você retire o transform de dentro do transition, deixando apenas o tempo de duração da animação, a animação ainda será feita porque o valor default, dado quando não é colocado nenhuma propriedade, executa todas as propriedades de transition.

Se você colocar para mudar a cor da div.child enquanto transform estiver dentro de transition, como no caso acima, haverá uma transição dura de cor, assim que o transform for acionado a cor mudará instanteneamente. No entanto, se não colocarmos o transform dentro de transition ocorrerá uma transição suave de cores, veja:

Você pode selecionar as propriedades que você quer que sofram uma transição suave, o restante sofrerá uma transição instantânea.

Por quê não usar o transition no :hover?

Se você colocar o transition dentro de um :hover a transição só ocorrerá enquanto o objeto em que o :hover está aplicado tiver o mouse sobre ele, caso o mouse seja tirado durante a transição o bloco que sofre a transição voltará para sua posição inicial de forma brusca. Veja:

Outras coisas que podemos especificar na transição é o delay e o timing-function. Onde:

  • timing-function: determina como a sua transição será executada;
  • delay: quanto tempo leva para a animação ocorrer após sua ativação;

Transition: timeing-function

Essa propriedade possui alguns valores pre-determinados, como o linear (padrão, se move em velocidade constante de um lado para o outro) e o ease-in-out (o inicio e fim do trajeto do objeto é devagar, o espaço entre eles o objeto vai na velocidade padrão).

Além desses valores você também pode criar uma forma,para isso basta você inspecionar a página, ver a propriedade transition de seu bloco e clicar em um quadrado com uma imagem que se encontra nele, lá você pode definir a velocidade durante a trajetória desse objeto, os valores dessa transição serão representados com cubic-bazier(1, 0.03, 0.56, 1.36) (exemplo), veja:

Transition: delay

Essa propriedade, como dito anteriormente, serve para dar um atraso na animação, dessa forma ela não será executada no exato momento de sua ativação. Veja:

Infelizmente não é possível fazer muito mais do que isso com a propriedade transition, é aí que entra a propriedade animation, onde você pode trabalhar com várias etapas.

Propriedade Animation:

A propriedade animation possui muitas mais opções do que a propriedade transition. Além disso sua sintaxe é mais simples do que a de transition. Por exemplo, para realizar um movimento da esquerda para a direita, em ambos os casos, veja:


            /* Transition */
            .child {
            transform: translateX(100%);
            }
            .parent:hover .child {
                transition: transform 1s ease-in;
            }

            /* Animation */

            .parent:hover .child {
                animation: left-to-right 1s ease-in;
            }

A estrutura da propriedade animation é relativamente mais simples, não acha? No entanto ainda tem uma coisa faltando, para que uma animação funcione é preciso dar um nome a ela, é aí que o @keyframes entra em ação.

Você coloca ele no CSS normalmente e dê um o nome a sua animation (nesse caso left-to-right), dê um nome que faça sentido com o que ela faz. Dentro das chaves é onde a animation é feita, você pode determinar o comportamento da animação desde o início (0%) até seu fim (100%), podendo definir como será seu trajeto dentro desse perído. Veja:


            /* Animation */

            .parent:hover .child {
                animation: left-to-right 1s ease-in;
            }

            @keyframes left-to-right {
                100% {
                    transform: translateX(100%);
                }
            }

Caso você não adicione o valor de 0%, como default ele vai pegar os valores dados para a .child, veja um exemplo prático dessa vez:

Uma coisa que é possível notar nessa animação é que quando seu trajeto acaba ela teleporta novamente para o início, ao invés de ficar no final até que o mouse seja retirado de dentro da div.parent. Para corrigir isso existe uma propriedade chamada animation-fill-mode e ela possui as seguintes opções:

  • backwards: antes da animação começar, aplicar todas as propriedades do keyframe 0%, se estiver em default irá voltar a sua posição original;
  • forwards: manter as propriedades do keyframe 100% após o término da animação enquanto sua condição de ativação (:hover) estiver ativa;
  • both: aplicará as propriedades tanto do keyframe 0% quanto do keyframe 100%, até que sua condição de ativação (:hover) seja disfeita;
  • none: após o término da animação nenhuma das propriedades dos keyframes será aplicada ao objeto;

Uma coisa que muitas pessoas não pensam quando se trata de animações css é performance, por isso tenha cuidado para não fazer uma animação ruim.

Vamos melhorar um pouco as coisas, que tal, ao invés de fazermos a div ir apenas da esquerda para direita, fazermos ela ir da esquerda para direita, baixo, esquerda, cima, voltando a sua posição original?

Para fazer essa animação foi colocado 5 atapas no keyframes, em cada etapa o objeto da animação ia para uma posição específica e alterava sua cor, definimos que essa animação ocorrerá infinitamente (usando a propriedade animation-iteration-count) até que sua condição de ativação seja desfeita, esse foi o código utilizado:


            .parent {
                position: absolute;
                top: 50%;
                left: 50%;
                transform: translate(-50%, -50%);
                margin: 0 auto;
                background-color: hsla(200, 100%, 20%);
                height: 250px;
                width: 250px;
            }

            .child {
                background-color: red;
                height: 50%;
                width: 50%;
            }

            .parent:hover .child {
                animation: move-around 1s ease-in forwards infinite;
            }

            @keyframes move-around{
                0% {
                    background-color: aqua;
                    transform: translateX(0%);
                }
                25% {
                    transform: translateX(100%);
                    background-color: purple;
                
                50% {
                    transform: translate(100%, 100%);
                    background-color: orange;
                
                75% {
                    transform: translate(0%, 100%);
                    background-color: darkred;
                }
                100% {
                    transform: translateY(0%);
                    background-color: magenta;
                }
            }
