Colocando o site no ar

Vamos usar o GitHub Pages para poder hospedar gratuitamente nosso site com ajuda do github.

Primeiro você vai abrir o repositório que você deseja hospedar gratuitamente para poder mostrar para os coleguinhas.

Procure o arquivo/pasta que você deseja hospedar dentro desse repositório.

Clique na opção "Settings" que é exibida abaixo do nome do repositório e de suas avaliações, etc.

Procure por "GitHub Pages", clique nele. Após isso clique na opção "Choose a theme" e selecione o tema que preferir, clique em "Select Theme" após fazer a escolha.

Agora você tem o endereço de sua hospedagem, que no meu caso foi: https://v4r3us1n.github.io/html5-css3/. Para ver o link de sua hospedagem basta ir novamente em "GiHub Pages".

O que será mostrado nesse link será o ".README" que foi criado anteriormente. Para você poder acessar os arquivos por esse link basta você colcar os links de cada exercício no ".README"

Para colocar um site no ar em GitHub Pages você, ao invés de clicar em "Select Theme", vai clicar na opção "None" e vai escolher a opção "main".

Agora esse seu site ganhou um link, o meu, por exemplo, é: https://v4r3us1n.github.io/projeto-android/

Você pode mandar esse link para qualquer um!

Você pode pegar esse seu link, ir no site "https://www.qr-code-generator.com/", clica na opção "URL" e coloca o endereço de seu site. Dessa forma você vai criar um QR Code para acessar seu site.

No entanto essa hospedagem tem algumas limitações. Ele só serve para hospedar sites que contenham HTML, CSS e JS na página. Ele não tem compatibilidade com Banco de Dados, PHP, Node JS, Python, etc.

Manutenção em sites hospedados no GitHub Pages

No caso do projeto-android, o parágrafo em seu header possuí uma margim-bottom que acaba criando uma barra horizontal no menu dependendo do tamanho da tela do dispositivo que acessa o site.

Para corrigir esse problema foi simples, bastou converter o "margim-bottom" em "padding-bottom".

Após fazer as alterações você tem que passar essa versão atualizada, como um commit, para o seu GitHub :)

Clonando Repositórios GitHub

Quando você vê um código muito bom é quer entender exatamente como ele funciona, ao invés de ter que refazer ele você pode simplesmente colná-lo no GitHub e mandá-lo para sua máquina local. Como fazer isso? Segue o pai...

Para clonar um respositório, ele tem que ser público para você poder clonar, basta você abrir o repositório e ir na opção "Code", lá você pode escolher abrir ele com o GitHub Desktop ou fazer download dele como arquivo ZIP. Recomendo a primeira opção... clicando na primeira opção você vai ter que autorizar abrir no GitHub Desktop.

Você vai poder escolher a pasta em que você quer colocar o repositório. Lembrando que não basta você apenas ler um repositório clonado que você entendeu, você tem que praticar também.
