Métodos Get e Post para Formulário

Vamos ver aqui 2 métodos de envio de formulários, sendo eles "get" e "post".

Na últime aula vimos que quando inserimos nossos dados no formulário e o enviamos, os mesmos ficam expostos no endereço do site. Pode conferir no formulário acima, é só olhar após o ".php?" da url.

Não é um problema eles estarem expostos assim? Sim, é, mas temos algumas soluções para isso querido gafanhoto!

Existem métodos mais avançados, que precisam de linguagens de programação, mas vamos focar só no HTML/CSS ok?

No HTML existem 2 métodos de envio de formulários, o "padrão", que é o método GET, e o método POST que oculta os dados enviados na url, mas eles ainda estão ali, só não estão sendo jogados na sua cara.

Existe uma maneira de ainda ver essas informações, que é você inspecionar a página, escolher a aba superior "Network", e digitar os dados no site. Se você olhar em Network você vai acessar a última chamada que foi realizada, vai em "Headers", lá será exibida a URL completa, incluindo os dados que foram "ocultados" pelo método POST.

O POST não protege seus dados, para você protegê-los você precisa criptografá-los, e para isso você precisa usar o HTTPS, um protocólo mais seguro do que o que estamos utilizando, que é o HTTP.

Quando Usar GET e POST

Você pode usar o GET quando os dados a serem inseridos não forem sensíveis, como Nome, Idade, etc. Você também pode utilizá-lo quando você quiser que o link seja compartilhável, como uma pesquisa no google, etc.

OBS: quando você envia dados usando o método GET você pode enviar até 3000 bytes (3000 letras), se o valor for superior a isso então sugiro que use outro método :T.

Você NÃO deve usar o GET quando os dados forem sensíveis ou quando for enviado através do formulário Fotos, envio de arquivos, etc.
