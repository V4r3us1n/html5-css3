Select, Datalist e Textarea em HTML

Agora nós vamos ver outros elementos que vão no formulário e que não são input.

Select

Imagine uma situação onde você quer que o usuário insira o estado onde ele mora, seria bem interessante que ele clicasse ali e visse a lista com os estados para ele escolher. Nesse caso usamos o select.

A tag select precisa de um name e id assim como o input, além disso, diferente do input, ele é uma tag que precisa ser fechada (<select></select>).

Dentro do select nós precisamos colocar opções para o usuário escolher, essas opções são inseridas nele através da tag option que também é uma tag que precisa ser fechada. Além disso, nela você precisará colocar um value para cada option e o nome que será exibido no select. Ex:


                <select>
                    <option value="SP">São Paulo</option>
                    <option value="RJ">Rio de Janeiro</option>
                </select>

Veja agora como ele aparece na tela do usuário:

Você também pode formar grupos entre os options, nesse caso você poderia separar eles por região. Para fazer isso nós utilizamos o optgroup.

No optgroup você pode colocar um label que serve como atributo, diferente dos outros casos que vimos até agora onde ele era usado como elemento.

Ficaria mais ou menos assim no seu código:

<select> <optgroup label="Região Sudeste"> <option value="SP">São Paulo</option> <option value="RJ">Rio de Janeiro</option> </optgroup> <optgroup label="Região Sul"> <option value="SC">Santa Catarina</option> <option value="RS">Rio Grande do Sul</option> </optgroup> </select>

Caso você queira que uma opção específica já venha selecionada basta você adicionar selected dentro dessa option.

Outra coisa que é comum ocorrer é colocar um option vazio pré selecionado.

Tá, mas e se eu quiser digitar outro estado que não esteja na lista?

Nesse caso, vamos para a próxima tag...

Datalist

Para fazer um datalist você vai precisar de um input do tipo text, nele você vai colocar o atributo list com o valor correspondente ao id do datalist. Ele ficaria dessa forma no seu código:


            <label for="idprof">Profissão: </label>
            <input type="text" name="prof" id="idprof" list="lprof">
            <datalist id="lprof">
                <option value="ADM">Administrador</option>
                <option value="CONT">Contador</option>
                <option value="DEV">Desenvolvedor</option>
                <option value="prof">Professor</option>
            </datalist>

Ele apareceria dessa forma para o usuário:

Caso você tire o value das options, não será mais exibido o valor que aparece sobre a profissão. Não é algo que você deva se preocupar, apenas deixe o value ali mesmo.

"Mas qual a diferença entre ele e o select? Bom, caso não tenha sua profissão, nesse exemplo, você pode adicionar ela ao datalist, caso você tenha que preencher novamente o formulário sua profissão já estará lá. Vale a pena ressaltar que isso só vale para você, se outra pessoa colocar outra opção em outro dispositivo ele não vai aparecer para você, entendeu?

Textarea

A tag textarea é usada quando você quer que o usuário digite algo de forma livre, nessa tag você deve informar o name, id, número de colunas que serão exibidas na tela (col) e o número de linhas que serão exibidas na tela (rol).


            <label for="imensagem">Mensagem: </label>
            <textarea name="mensagem" id="imensagem" cols="30" rows="10"></textarea>

Ele aparece da seguinte forma para o usuário:

Vale a pena ressaltar que o nº de linhas e colunas que você coloca no textarea não limita o quanto o usuário pode digitar. Você também consegue alterar o tamanho da caixa de texto segurando e puxando o canto inferior direito da caixa de texto.

Quando você for usar textarea no seu formulário, é recomendado que você use no formulário o method="post", para não ter problema com tamanho limite que nem o método get(padrão).

Outra coisa que vale citar é que não funciona placeholder no textarea, a única forma de colocar uma mensagem dentro dele é se você colocar a mensagem dentro da tag dele, mas o usuário teria que apagar essa mensagem quando quisesse escrever algo, isso não seria muito legal por isso não coloque mensagens dentro do textarea.
