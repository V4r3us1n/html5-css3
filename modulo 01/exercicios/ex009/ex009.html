Ligações em Toda Parte

O link/ligação é a essência fundamental do hipertexto. Ela permite ligar documentos entre si e permitir a navegação entre essas páginas.

Jogue a âncora

Os hyperlinks são um dos conceitos mais antigos da HTML. Ele permite que você lingue um ponto a outro na World Wide Web. Até os mecanismos de busca utilizam dos hyperlinks de um site, para o Google achar determinado site ele fica vasculhando outros sites, procurando o link dele.

Para criar um hyperlink devemos criar âncoras através da tag <a>. O principal atributo dessa tag é o href, que cria uma referência hipertexto. Ex:
Acesse o pdf dessa aula

Dentro do atributo "href" colocamos uma URL inteira, que te leva para outro site.

Outro atributo útil da tag de âncora é o hreflang, que permite indicar o idioma principal do site ligado ao seu link. Ex:
Clica aí para ganhar cristais grátis no firo firo

Mira no Alvo

O padrão desses hyperlinks é que, ao clicar neles, o site de destino abre na mesma janela do site, o que faz que o conteúdo anterior deixe de ser exibido.

Isso é bom quando se trata de abrir documentos do mesmo site, pois eles continuam sendo do mesmo site.
Mas isso pode ser um problema quando o hyperlink leva os visitantes para outro site, e se eles nunca mais voltarem?

Para poder controlar onde o site de destino vai abrir usamos o atributo target, que suporta os valores a baixo:

  • _blank vai abrir o link em uma nova janela em branco
  • _self vai abrir o link na janela atual (padrão)
  • _top desfaz todos os frames e abre o destino no navegador completo
  • parent similar ao uso do _top em uma referência à janela mãe
  • nome-do-frame caso esteja usando frames, indicar o nome da janela a abrir

Como o uso de frames está praticamente em desuso, vamos usar só as duas primeiras opções _blank e _self. Ex: Abrir link na janela atual
Abrir link em nova janela

Esse link é seu ou dos outros?

Você pode indicar qual é a natureza do destino usando o atributo rel. Alguns dos valores que ele aceita são:

  • next : o link é para a próxima parte do documento atual
  • prev : o link é para a parte anterior do documento atual
  • author : o link é para o site do autor do artigo atual
  • external : o link é para outro site que não faz parte do site atual
  • nofollow : o link é para um site não endossado, como um link pago

Ex:
Continua navegando no site
Abre perfil GitHub em nova janela

Caso você queira acessar um arquivo que esteja em uma subpasta você precisa colocar o caminho, ou então usar "../" antes do nome do arquivo.

No código anterior, o primeiro link é o que chamamaos de link local ou link interno. Leva o visitante para outro página dentro do mesmo site, nesse caso não é necessário indicar a URL completa.
Já o segundo link nos leva a outro site, o que chamamos de link externo. Nesse caso devemos indicar a URL completa, incluindo o protocolo, seja ele http:// ou https://, também precisa indicar o caminho que leva à uma página específica, caso necessário.

Navegando por pastas locais

Servidores Web normalmente rodam o sistema operacional Linux. Se estiver se referindo à pasta atual do servidro use "./" antes do nome do arquivo. Caso seja à pasta imediatamente superior na hierarquia, use "../" para voltar um nível para a pasta mais externa. E por favor, Aprenda Linux.

E para fazer Downloads?

É comum aparecer em sites links para efetuar downloads de arquivos, sejam eles PDF, arquivo ZIP, ou qualquer outro. A partir da versão HTML5, as âncoras recebem atributos especiais para isso. Basta fazer o link diretamente para o arquivo que se deseja baixar e adicionar o atributo download com o valor configurado para o nome do arquivo a ser baixado e o atributo type para indicar ao navegado que tipo de arquivo está sendo baixado. Ex:
Baixe aqui o PDF do meu livro

Alguns exemplos de media types usados nos downloads são:

  • application/pdf
  • application/zip
  • text/html
  • text/css
  • text/javascript
  • video/mp4
  • video/H264
  • video/JPEG
  • audio/aac
  • audio/mpeg
  • font/ttf
  • image/jpeg
  • image/png
